## Parameter Descriptions

### `response_format`

**Short Description:** Response format (json_object, json_schema) - structured output

**Type:** `map (object)`

**Default:** None

**Range:** json_object, json_schema

**Details:** Forces the model's output format. Setting `{ "type": "json_object" }` enables JSON mode, which guarantees that the message generated by the model is valid JSON. The `json_schema` mode allows you to specify a strict JSON schema for structured output.

---

### `tools`

**Short Description:** List of available functions for the model to call (tool calling)

**Type:** `array`

**Default:** None

**Range:** array of function objects

**Details:** Allows the model to call external functions. The model analyzes the request and decides which functions to call with which parameters. Each function is described with its name, description, and parameters in JSON Schema format. The model returns a structured function call that is then executed in your application, and the result is passed back to the model.

---

### `temperature`

**Short Description:** Generation temperature - response creativity

**Type:** `float`

**Default:** 1.0

**Range:** 0.0 to 2.0

**Details:** This parameter affects the diversity of model responses. Low values lead to more predictable and typical responses, while high values promote more diverse and less common responses. At a value of 0, the model always gives the same response for a given input.

---

### `top_p`

**Short Description:** Nucleus sampling - alternative to temperature

**Type:** `float`

**Default:** 1.0

**Range:** 0.0 to 1.0

**Details:** Controls response diversity through nucleus sampling. The parameter determines the minimum probability of tokens that will be considered. For example, top_p=0.1 means that only tokens with probability >= 10% are considered. Used as an alternative or together with temperature for more precise generation control.

---

### `reasoning`

**Short Description:** Enable reasoning mode (thinking mode)

**Type:** `boolean/enum`

**Default:** false

**Range:** varies by model

**Details:** Enables reasoning mode (thinking mode) for models that support it. This allows the model to perform multi-step reasoning before generating the final answer. Some models support this feature for advanced reasoning tasks.

---

### `frequency_penalty`

**Short Description:** Token frequency penalty

**Type:** `float`

**Default:** 0.0

**Range:** -2.0 to 2.0

**Details:** This parameter aims to control token repetition based on how often they appear in the input data. It tries to use less frequently those tokens that appear more often in the input data, proportionally to their frequency of occurrence. The token penalty scales with the number of occurrences. Negative values will encourage token reuse.
