Легенда ​

Параметры модели ​

  • 📋 — response_format (Формат ответа (json_object, json_schema) - структурированный вывод)
  • 🔧 — tools (Список доступных функций для вызова моделью (tool calling))
  • 🌡️ — temperature (Температура генерации - креативность ответов)
  • 📊 — top_p (Nucleus sampling - альтернатива temperature)
  • 🧠 — reasoning (Включение режима рассуждения (thinking mode))
  • 🔁 — frequency_penalty (Штраф за частоту использования токенов)

Модальности ​

Входные модальности (что модель может принимать):

  • 📝 — текст (text)
  • 🖼️ — изображение (image)
  • 🎬 — видео (video)
  • 🎵 — аудио (audio)
  • 📎 — файл (file)

Выходные модальности (что модель может возвращать):

  • 📝 — текст (text)
  • 🖼️ — изображение (image)
  • 🔍 — векторные представления (embeddings)

Ценообразование ​

Цены указаны за миллион токенов в формате ₽prompt · ₽completion · ₽image · ₽video · ₽request · ₽web_search (в рублях):

  • ₽prompt — цена за входные токены (prompt)
  • ₽completion — цена за выходные токены (completion)
  • ₽image — цена за обработку изображений
  • ₽video — цена за обработку видео
  • ₽request — цена за запрос
  • ₽web_search — цена за веб-поиск

Если цена недоступна или равна 0, указывается -. Цены отображаются в компактном формате:

  • k — тысячи (например, ₽1.3k = 1,300 рублях)
  • m — миллионы (например, ₽618m = 618,000,000 рублях)

Описание параметров ​

response_format ​

Краткое описание: Формат ответа (json_object, json_schema) - структурированный вывод

Тип: map (object)

По умолчанию: None

Диапазон: json_object, json_schema

Подробности: Принудительно задает формат выходных данных модели. Установка { "type": "json_object" } включает JSON режим, который гарантирует, что сообщение, сгенерированное моделью, является валидным JSON. Режим json_schema позволяет задать строгую JSON схему для структурированного вывода.

tools ​

Краткое описание: Список доступных функций для вызова моделью (tool calling)

Тип: array

По умолчанию: None

Диапазон: array of function objects

Подробности: Позволяет модели вызывать внешние функции. Модель анализирует запрос и решает, какие функции вызвать с какими параметрами. Каждая функция описывается с указанием имени, описания и параметров в формате JSON Schema. Модель возвращает структурированный вызов функции, который затем выполняется в вашем приложении, а результат передается обратно модели.

temperature ​

Краткое описание: Температура генерации - креативность ответов

Тип: float

По умолчанию: 1.0

Диапазон: 0.0 to 2.0

Подробности: Этот параметр влияет на разнообразие ответов модели. Низкие значения приводят к более предсказуемым и типичным ответам, а высокие значения способствуют более разнообразным и менее распространенным ответам. При значении 0 модель всегда дает один и тот же ответ для заданного ввода.

top_p ​

Краткое описание: Nucleus sampling - альтернатива temperature

Тип: float

По умолчанию: 1.0

Диапазон: 0.0 to 1.0

Подробности: Контролирует разнообразие ответов через nucleus sampling. Параметр определяет минимальную вероятность токенов, которые будут рассматриваться. Например, top_p=0.1 означает, что рассматриваются только токены с вероятностью >= 10%. Используется как альтернатива или вместе с temperature для более точного контроля генерации.

reasoning ​

Краткое описание: Включение режима рассуждения (thinking mode)

Тип: boolean/enum

По умолчанию: false

Диапазон: varies by model

Подробности: Включает режим рассуждения (thinking mode) для моделей, которые его поддерживают. Это позволяет модели выполнять многошаговые рассуждения перед генерацией окончательного ответа. Некоторые модели поддерживают эту функцию для продвинутых задач рассуждения.

frequency_penalty ​

Краткое описание: Штраф за частоту использования токенов

Тип: float

По умолчанию: 0.0

Диапазон: -2.0 to 2.0

Подробности: Этот параметр направлен на контроль повторения токенов на основе того, как часто они появляются во входных данных. Он пытается использовать реже те токены, которые появляются чаще во входных данных, пропорционально частоте их появления. Штраф токена масштабируется с количеством вхождений. Отрицательные значения будут способствовать повторному использованию токенов.
