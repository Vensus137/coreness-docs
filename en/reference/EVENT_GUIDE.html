Event Types ‚Äã

System supports following event types:

Event TypeDescriptionWhen Occurs
messageRegular message from userUser sends text message or media
callbackInline button clickUser clicks button in message
member_joinedMember joined groupUser joins group or supergroup
member_leftMember left groupUser leaves group or supergroup
pre_checkout_queryPayment confirmation requestUser clicks "Pay" in invoice, but before payment processing
payment_successfulSuccessful paymentPayment via invoice successfully processed

Common Fields ‚Äã

These fields available in all event types. All these fields come initially when receiving event and contain original data that formed the event.

‚ÑπÔ∏è Important: These attributes ‚Äî original event information passed from source (e.g., Telegram). They used by system as default values for actions (e.g., chat_id and message_id taken from event if not specified explicitly in action parameters).

‚ö†Ô∏è Limitation for scheduled scenarios: In scenarios launched on schedule (with schedule field), most common fields unavailable, as event not associated with user or chat. Only system fields available (bot_id, tenant_id) and special scheduled scenario fields (see Scheduled Scenario Fields section).

üîß Service Fields ‚Äã

System automatically adds following service fields for correct action operation:

FieldTypeDescriptionPurpose
bot_idintegerBot IDRequired for sending messages and executing actions
tenant_idintegerTenant IDRequired for loading scenarios and determining context
polling_start_timestringBot polling start time (ISO)Service for filtering events by time
last_errorstringLast action error textService for passing errors between scenario steps
last_resultstringLast executed action resultService for debugging and result checking between scenario steps
scenario_chainarrayScenario chain in execution orderService for debugging and logging, contains array of scenario names in execution order

‚ö†Ô∏è Important: Following fields added automatically:

  • Required fields (bot_id, tenant_id) ‚Äî required for executing actions (sending messages, working with DB, managing states)
    • ‚ÑπÔ∏è In scheduled scenarios: Fields bot_id and tenant_id available automatically and don't require explicit specification in action parameters. They automatically substituted from synthetic event context
  • Service fields (polling_start_time, last_error, last_result, scenario_chain) ‚Äî used by system for internal logic:
    • polling_start_time ‚Äî filtering events by time
    • last_error ‚Äî saving action error text for subsequent scenario steps
    • last_result ‚Äî saving action execution result (e.g., success, error, timeout, not_found) for debugging and checking in subsequent scenario steps
    • scenario_chain ‚Äî array of scenario names in call order, automatically updated when launching each scenario and on transitions (jump_to_scenario, execute_scenario). Current scenario available as scenario_chain[-1], previous ‚Äî as scenario_chain[-2] (if exists). Used for debugging and logging execution flow
  • State fields (user_state, user_state_expired_at) ‚Äî can be null if state not set or expired

Payment Events ‚Äã

Events related to payment via Telegram invoices.

pre_checkout_query Fields ‚Äã

Event pre_checkout_query occurs when user clicks "Pay" button in invoice, but before payment processing. This is payment confirmation request from Telegram to bot.

‚ÑπÔ∏è Important: Must respond to this event via answer_pre_checkout_query action to confirm or reject payment. If don't respond within 10 seconds, payment will be automatically rejected.

FieldTypeDescriptionExample
pre_checkout_query_idstringPayment confirmation request ID"1234567890123456789"
invoice_payloadstringInvoice ID (payload passed during creation)"123"
currencystringPayment currency"XTR" (Telegram Stars)
total_amountintegerTotal payment amount in minimal currency units100 (100 stars)

Additionally: Event contains common fields (see Common Fields section).

payment_successful Fields ‚Äã

Event payment_successful occurs after successful payment processing via invoice.

FieldTypeDescriptionExample
invoice_payloadstringInvoice ID (payload passed during creation)"123"

total_amountintegerTotal payment amount in minimal currency units100 (100 stars)
telegram_payment_charge_idstringPayment ID in Telegram"1234567890"

Additionally: Event contains common fields (see Common Fields section).

Member Events ‚Äã

Events of type member_joined and member_left use standard common fields (see Common Fields section).

‚ÑπÔ∏è Important:

  • Event member_joined occurs when user joins group (transition from status left, kicked or first join)
  • Event member_left occurs when user leaves group (transition to status left or kicked)
  • These events available only for groups and supergroups

Usage Examples ‚Äã

How to use event fields in scenarios:

Simple Conditions ‚Äã

yaml
# Check event type
trigger:
  - event_type: "message"

# Check message text
trigger:
  - event_type: "message"
    event_text: "/start"

# Check callback data
trigger:
  - event_type: "callback"
    callback_data: "menu_main"

# Check user state
trigger:
  - event_type: "message"
    user_state: "feedback"

Complex Conditions ‚Äã

yaml
# Check user and chat
trigger:
  - event_type: "message"
    condition: |
      $user_id == 123456789 and
      $chat_type == "private" and
      $event_text ~ "hello"

# Check attachments
trigger:
  - event_type: "message"
    condition: |
      $event_attachment[0].type == "photo"

# Check reply message
trigger:
  - event_type: "message"
    condition: |
      $is_reply == true and
      $reply_user_id == 987654321

Usage in Action Parameters ‚Äã

yaml
# Send message to same chat
- action: "send_message"
  params:
    bot_id: "{bot_id}"
    chat_id: "{chat_id}"
    text: "Hello, {first_name}!"

# Reply to message
- action: "send_message"
  params:
    bot_id: "{bot_id}"
    chat_id: "{chat_id}"
    text: "Received your message!"
    message_reply: "{message_id}"

# Manage user state
- action: "set_user_state"
  params:
    state: "feedback"
    expires_in_seconds: 3600
